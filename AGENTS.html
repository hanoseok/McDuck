McDuck Agent Guide

이 파일은 McDuck 저장소의 기본 작업 가이드입니다. 장기적으로 유지할 규칙과 명령은 이 HTML 파일에 정리합니다.

기본 원칙

  • 작업을 시작하기 전에 이 파일과 현재 git 상태를 확인합니다.
  • 사용자가 요청하지 않은 리팩터링, 포맷 변경, 파일 이동은 피합니다.
  • 기존 사용자 변경을 되돌리지 않습니다. 충돌이 있으면 먼저 상태를 설명합니다.
  • 검증 없이 완료를 주장하지 않습니다. 변경 후 관련 테스트와 빌드를 실행합니다.

프로젝트 개요

McDuck은 bunx ccusage 출력을 읽어 macOS 상태바 팝오버에서 LLM token 사용량을 보여주는 네이티브 앱입니다.

  • 앱 타깃: Sources/McDuck
  • 테스트 가능한 코어 로직: Sources/McDuckCore
  • 코어 테스트: Tests/McDuckCoreTests
  • 앱 번들 스크립트: scripts/build-app.sh

기술 스택

  • Swift Package Manager
  • SwiftUI MenuBarExtra
  • Swift Charts
  • Liquid Glass: GlassEffectContainer, glassEffect
  • Swift Testing

빌드와 검증

기능 변경 후 최소한 아래 명령을 실행합니다.

swift test
swift build
scripts/build-app.sh

패키징 결과는 dist/McDuck.app에 생성됩니다. dist/는 git에 포함하지 않습니다.

MCDUCK_VERSION(예: 1.2.0)을 설정하면 번들의 CFBundleShortVersionString에, MCDUCK_BUILD는 CFBundleVersion에 기록됩니다.

릴리스와 버저닝

버전 태그를 push 하면 .github/workflows/release.yml이 macOS 러너에서 앱을 빌드해 GitHub Release로 게시합니다.

git tag v1.2.0
git push origin v1.2.0
  • 트리거: v* 형식 태그 push (예: v1.2.0).
  • 워크플로는 swift test 후 scripts/build-app.sh로 빌드하고 태그 버전을 번들에 스탬프합니다.
  • .app은 ditto로 McDuck-<tag>-macos.zip으로 압축하고 .sha256 체크섬과 함께 Release에 첨부합니다.
  • 다운로드: 저장소 Releases 페이지 또는 gh release download <tag>.
  • 버전 태그는 의미 있는 버전(semver)을 사용하고, 한 번 게시한 태그는 다시 사용하지 않습니다.

의존성 처리

  • 앱은 시작 시 Bun과 ccusage 실행 가능 여부를 검사합니다.
  • 의존성이 없으면 자동 설치하지 않고, 사용자가 팝오버의 설치 버튼을 눌렀을 때만 진행합니다.
  • ccusage는 글로벌 설치보다 bunx ccusage 실행을 우선합니다.

구현 규칙

  • JSON 파싱, 명령 실행, 상태 판정은 McDuckCore에 둡니다.
  • SwiftUI 뷰와 macOS 앱 진입점은 Sources/McDuck에 둡니다.
  • 코어 동작을 바꾸는 경우 테스트를 먼저 추가하거나 갱신합니다.
  • UI는 상태바 유틸리티에 맞게 작고 명확하게 유지합니다.
  • macOS 최신 API를 사용할 때는 빌드 가능한 fallback 또는 availability check를 둡니다.

Git 규칙

  • 기능 작업은 codex/ 접두어 브랜치에서 진행합니다.
  • 의미 있는 단위로 커밋합니다.
  • push 전에 git status --short로 의도하지 않은 파일이 없는지 확인합니다.
  • 생성물, 캐시, 임시 브레인스토밍 파일은 커밋하지 않습니다.

문서 규칙

Canonical guide: 저장소 가이드는 이 AGENTS.html 파일에 정리합니다. CLAUDE.md는 이 파일을 읽으라는 포인터 역할만 합니다.

  • 새로운 장기 규칙은 이 파일에 추가합니다.
  • 도구별 지침을 중복 작성하지 않습니다.
  • README는 사용자용 설명, 이 파일은 작업 에이전트용 설명으로 유지합니다.
